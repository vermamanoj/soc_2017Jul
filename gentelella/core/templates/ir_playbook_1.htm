Log relevant information from suspect system

Research source domain/IP

Determine if the source is a known IT scanning server such as a vulnerability scanning system.

 

 

 

Known source

If a known source scanner application, whitelist the source and proceed to step number 5.2.

 

 

 

Unknown destination

Escalate to L2.

 

 

 

Identify scanning tools

Attempt to identify if any known scanning tools, such as NMAP or Nessus, are installed on the source device. This can be accomplished by analyzing asset inventory data such as Microsoft SCCM or by connecting to the source device. If known scanning tools are detected on the device, obtain all logs, scan configuration files and results available on the machine.

 

 

 

Analyze results

If step 2.1 determined that scanning tools are installed on the source, analyze the logs and scan reports to determine level of activity.

 

 

 

Refer incident to policy violation resources

If steps 2.1 and 2.2 indicate purposeful scanning by an end user, refer the incident to the internal parties responsible for policy violations.

 

 

 

Unknown scan source

If steps 2.1 and 2.2 do not indicate a known or user initiated scan, it should be assumed that this is malware-based activity.

 

 

 

Collect and analyze destination device logs

Collect all available logs related to the scan target. Analyze for exploitation of open ports on the device.

 

 

 

Remove source machine from network

If activity was not user generated, open an IT support ticket to remove the system from the network.

 

 

 

Image target device

If exploitation of the target is confirmed, obtain and analyze the system image.

 

 

 

Reimage source device

If activity was not user generated, the system should be rebuilt. Open an IT Support ticket to have the device reimaged. If a critical asset, the incident may need to be escalated to determine the impact of such a reimage.

 

 

 

Analyze target image

Analyze image of target device for exploitation, including account compromise, malicious software that was dropped on the device and possible data exfiltration.

 

 

 

Determine target remediation

If target device is found to be compromised, reimage. If reimage is not possible due to the device type, define remediation plan with device owner.

 

 

 

Incident closure

If a user-based scan, obtain a summary from the policy violation resource (if available). Close incident, noting all activity.

 

 

 

Close and communicate

Ensure that incident details are documented, metrics are updated and the incident is communicated to Senior Management, if applicable.
